CS 300 Reflection

What was the problem you were solving in the projects for this course?

The primary problem I was solving in this course was determining how to efficiently store, organize, and retrieve course information for a computer science advising system. The projects required loading course data from a file, managing prerequisite relationships, searching for specific courses, and displaying all courses in alphanumeric order. To accomplish these tasks, I analyzed different data structures and implemented a solution that balanced performance and usability.

How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by first analyzing the strengths and weaknesses of several data structures, including vectors, hash tables, and binary search trees. Understanding data structures is important because they directly affect how efficiently data can be stored, searched, and maintained. By comparing run-time performance and memory usage, I was able to determine which structure would best support the program requirements and provide efficient course lookups and sorting capabilities.

How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges was correctly loading and processing the CSV file containing course information. I also encountered issues with file paths and debugging program execution in Visual Studio. I overcame these challenges by testing smaller portions of the program, reviewing error messages carefully, and validating the input data before processing it. Breaking the project into smaller tasks made troubleshooting much more manageable.

How has your work on this project expanded your approach to designing software and developing programs?

This project reinforced the importance of planning before writing code. Analyzing data structures before implementation helped me understand how design decisions affect performance and scalability. I learned that selecting the appropriate data structure early in development can simplify implementation and improve the overall effectiveness of a program.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This project encouraged me to write code using clear function separation, meaningful variable names, and modular program design. Organizing the code into specific functions for loading, searching, sorting, and displaying data made the program easier to read and maintain. I also gained a better appreciation for designing software that can be modified in the future without requiring major changes to existing code.
